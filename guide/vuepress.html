# vuepress

# 搭建个人日志

VuePress 是尤雨溪（vue.js 框架作者）4 月 12 日发布的一个全新的基于 vue 的静态网站生成器，实际上就是一个 vue 的 spa 应用，内置 webpack，可以用来写文档。

VuePress 有很多优点：

  • 界面简洁优雅（个人感觉比 HEXO 好看）
  • 容易上手（半小时能搭好整个项目）
  • 更好的兼容、扩展 Markdown 语法
  • 响应式布局，PC 端、手机端
  • oogle Analytics 集成
  • 支持 PWA

# 开始搭建

1.可以右键手动新建，也可以使用 mkdir 命令新建：

mkdir vuepressBlogDemo

2.安装 vuepress,不推荐全局安装

npm install -D vuepress

3.创建你的第一篇文档,也可以在 docs 目录下新建 README.md 文件，（这是默认文件，必须存在）

mkdir docs && echo '# Hello VuePress' > docs/README.md

4.添加脚本，可添加 --temp .temp 开启热加载功能（每次代码修改不用重启项目）。

{
  "scripts": {
    "dev": "vuepress dev docs",
    // "docs:dev": "vuepress dev docs --temp .temp", // 开启热加载
    "build": "vuepress build docs"
  }
}

5.启动

npm run dev

# 部署到 github

分为两步，第一步新建一个库用来显示日志，第二部用来存储更新的日志内容。

1.登录GitHub (opens new window),打开 github 网站，登陆自己的 github 账号（没有账号的快去注册并面壁思过作为一个优秀的程序员为啥连一个 github 账号都没有） 接着我们新建两个仓库

# 新建仓库一： USERNAME.github.io （不用克隆到本地）

注意！ USERNAME 必须是你 Github 的账号名称，不是你的名字拼音，也不是你的非主流网名，不要瞎起，要保证和 Github 账号名一模一样！

例如我的 Github 账号名称是 helloworldtang 那么新建仓库，Repository name 就填写为：helloworldtang.github.io 这个仓库建好后，不用克隆到本地，内容更新修改都在下面的仓库中进行。

# 新建仓库二：随便起一个名字，比如：vuepressBlog （克隆到本地）

这个项目是用来开发博客的，以后只需要改这个项目就够了。 将 vuepressBlogDemo 文件夹的内容拷贝到仓库二

#!/usr/bin/env sh

# 确保脚本抛出遇到的错误
# set -e

# 生成静态文件
npm run build

# 进入生成的文件夹
cd docs/.vuepress/dist

# 如果是发布到自定义域名
# echo 'www.yourwebsite.com' > CNAME

git init
git add -A
git commit -m 'deploy'

# 如果你想要部署到 https://USERNAME.github.io
# 这个是 ssh 模式git@github.com:USERNAME/USERNAME.github.io.git， 如果是https 克隆的需要使用
# https://github.com/USERNAME/USERNAME.github.io.git

# git push -f git@github.com:USERNAME/USERNAME.github.io.git master
# 我是用https 克隆的，所有用下面这个
git push -f https://github.com/USERNAME/USERNAME.github.io.git master

# 如果发布到 https://USERNAME.github.io/<REPO>  REPO=github上的项目
# git push -f git@github.com:USERNAME/<REPO>.git master:gh-pages

cd -

# 修改仓库二中的 deploy.sh 发布脚本

把文件中的 USERNAME 改成 Github 账号名，例如我的账号名是 zhangyunchencc，那么就可以改为：

# 如果你想要部署到 https://USERNAME.github.io
git push -f git@github.com:zhangyunchencc/zhangyunchencc.github.io.git master

这样仓库二和仓库一就建立了关联。

简单说二者的关系是：仓库一负责显示网站内容，我们不需要改动它；日常开发和新增内容，都在仓库二中，并通过 npm run deploy 命令，将代码发布到仓库一。

"scripts": {
  "deploy": "bash deploy.sh"
}

最后运行 npm run deploy （将仓库二中 dist 代码部署到仓库一中）。

此时打开 Github Settings 中下面的链接: https://zhangyunchencc.github.io/ 即可看到自己的主页啦~
